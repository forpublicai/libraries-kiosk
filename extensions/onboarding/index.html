PublicPass Browser Extensions

Onboarding Guide for Library Administrators (Chromebooks)

1. What PublicPass Does

PublicPass is a pair of browser extensions that make it easy and safe to log patrons into websites (like research databases, learning platforms, or other online services) on shared library devices.

Admin Extension (for staff)

  • Lets you log in once as an administrator.
  • Captures that signed‚Äëin ‚Äúsession‚Äù securely.
  • Sends it to a patron‚Äôs Chromebook with a few clicks.

User Extension (for patrons)

  • Receives the session from the admin.
  • Applies the login to the patron‚Äôs browser.
  • Can automatically log the patron out later, so sessions don‚Äôt linger.

Think of it as a secure ‚Äúremote login handoff‚Äù from staff to patron, designed for shared Chromebooks.

2. Key Features

No Passwords on Patron Devices

  • Patrons never see or type the library‚Äôs shared passwords.
  • Sessions are delivered already logged-in, via the extension.

Secure Session Sharing

  • Sessions are encrypted end‚Äëto‚Äëend between admin and user extensions.
  • Only the intended user device can decrypt and use a session.

Automatic Logout (Session Time Limits)

  • Admins can set how long a session lasts.
  • After the time is up, the extension can:
    • Clear cookies and storage for that site.
    • Close the tab or log out, depending on configuration.
  • This protects patrons‚Äô privacy on shared Chromebooks.

Per-Website / Per-Tab

  • Admins log in only to the websites they want to share.
  • Sessions are captured per site, not for your entire browser.

Cross-Device Compatible

  • Works between different Chromebooks.
  • Designed for shared and rotating devices, like public library terminals or checkout Chromebooks.

3. What You Need

Devices

  • Staff: A Chromebook with the Admin Extension installed.
  • Patrons: Chromebooks with the User Extension installed.

Browser: Google Chrome (or the Chrome browser on Chromebooks).

Access

  • Internet access for both staff and patron devices.
  • Access to the online services you want to share (e.g., databases, learning tools).

4. Installing the Extensions

A. From the Chrome Web Store (recommended)

Do this on each device type.

1) Admin Extension (staff Chromebooks)

  1. Open Chrome.
  2. Go to the Chrome Web Store.
  3. Search for: ‚ÄúPublicPass Admin‚Äù.
  4. Click Add to Chrome ‚Üí Add extension.
  5. Pin it by clicking the puzzle icon (Extensions) in the toolbar and clicking the pin next to PublicPass Admin.

2) User Extension (patron Chromebooks)

  1. On a patron Chromebook, open Chrome.
  2. Go to the Chrome Web Store.
  3. Search for: ‚ÄúPublicPass User‚Äù.
  4. Click Add to Chrome ‚Üí Add extension.
  5. Pin it by clicking the puzzle icon (Extensions) and pinning PublicPass User.
Tip for managed Chromebooks: Your IT team can push these extensions to all staff/patron devices through the Google Admin Console, instead of installing them by hand.

B. If You Have ZIP Files (fallback method)

If your IT team provided .zip files instead of using the Web Store:

1) Prepare the ZIPs

  • You should have two ZIP files, for example:
  • admin-extension.zip
  • user-extension.zip

Unzip them somewhere on each Chromebook (or on a shared drive your IT can access).

2) Enable Developer Mode (per device)

This is usually done by IT. Only use this method if the Web Store is not an option.
  1. In Chrome, go to: chrome://extensions
  2. Turn on Developer mode (top right).
  3. Click Load unpacked.
  4. Select the unzipped folder for:
    • Admin device: admin-extension
    • Patron device: user-extension
  5. Pin the extension in the toolbar (via the puzzle icon).

5. First-Time Setup for Admins

Once the Admin Extension is installed on a staff Chromebook:

  1. Click the PublicPass Admin icon in the toolbar.
  2. Click Options or Settings (depending on the version).
  3. Set:
    • Server URL: This is usually pre‚Äëfilled. If IT gave you a URL, paste it here.
    • Admin Username: A name or identifier for your admin account (e.g., central-library-admin).
  4. Click Save.

The extension will:

  • Register your identity with the server (in the background).
  • Store a secure key so only you can send sessions as this admin.

You only need to do this once per admin.

6. First-Time Setup for User Devices

On a patron Chromebook:

  1. Click the PublicPass User extension icon.
  2. Open Options / Settings.
  3. Set:
    • Server URL: Same as the admin‚Äôs server URL (usually pre‚Äëfilled).
    • Username: A simple library identifier, such as branch-a-chromebooks.

    All patron devices at the same branch can share the same username, or you can set one per device‚Äîwhatever your policy prefers.

  4. Click Save.

The extension will register the device/user with the PublicPass server so admins can send sessions to it.

7. Everyday Workflow: How to Share a Session

Step 1 ‚Äì Admin logs into the website

On the staff Chromebook:

  1. Open Chrome.
  2. Go to the website you want patrons to use (for example, a database or service).
  3. Log in as usual with the library‚Äôs institutional or admin credentials.
  4. Make sure you are fully signed in and on the page you want patrons to land on.

Step 2 ‚Äì Admin shares the session

Still on the staff Chromebook:

  1. Click the PublicPass Admin extension icon.
  2. Choose:
    • Which patron/user to send to (you‚Äôll see usernames of registered user extensions).
    • Optionally, a comment (e.g., ‚ÄúReplit login for table 3‚Äù).
    • (If available) How long the session should last before auto‚Äëlogout.
  3. Click Share session (or similar language).

The extension will:

  • Capture the current login session (cookies + storage).
  • Encrypt it end‚Äëto‚Äëend.
  • Send it securely to the chosen user device.

Step 3 ‚Äì Patron accepts and uses the session

On the patron Chromebook:

  1. The PublicPass User extension shows a notification (or a small indicator) that a session has been received.
  2. The patron (or staff helping them) clicks:
    • The notification, or
    • The PublicPass User icon ‚Üí Check inbox / Accept session.
  3. The extension:
    • Applies the login to the browser.
    • Opens the target website already logged in.
    • Optionally schedules auto‚Äëlogout (if configured by the admin).

From the patron‚Äôs perspective, they are simply taken to the site and already signed in.

8. Good Practices for Library Staff

Use dedicated admin devices

Keep admin logins on staff Chromebooks; use user extensions on patron Chromebooks only.

Keep extensions up to date

  • The Chrome Web Store updates extensions automatically.
  • If using ZIPs, plan regular updates via IT.

Use session time limits

  • Set a reasonable session duration (e.g., 30‚Äì60 minutes).
  • This balances convenience with privacy and security.

Train staff with a simple script

  • ‚ÄúLog in as usual ‚Üí Click PublicPass Admin ‚Üí Choose patron ‚Üí Share.‚Äù
  • ‚ÄúPatron clicks PublicPass User ‚Üí Accept session ‚Üí Use site.‚Äù

Clear policies on who can send sessions

  • Limit admin extensions to staff accounts only.
  • Do not install the admin extension on patron devices.

9. Troubleshooting Tips

The patron doesn‚Äôt see a session

  • On the patron Chromebook, click the PublicPass User icon and press Check now.
  • Make sure the username on the user extension matches what the admin is sending to.
  • Confirm both devices have internet access.

Website opens but isn‚Äôt logged in

  • Ensure the admin was fully logged in before sharing.
  • Try sharing again from a fresh, logged‚Äëin tab.
  • If the site has strict security (e.g., unusual device checks), contact mohsin@publicai.co Some services may be less compatible with session transfer.

Sessions seem to stick around too long

  • Lower the default session time limit in the admin extension options.
  • Encourage staff to revoke sessions or use manual logout when needed.

Using ZIP versions and seeing warnings

  • Developer mode is normal for ZIP-based installs.
